# AGENTS

- Follow `CONTRIBUTING.md`, `.github/PULL_REQUEST_TEMPLATE.md`, `.github/workflows/pr-lint.yml`, and `.github/scripts/pr-validate-description.js` before making any PR-facing change.
- Keep changes focused and low-churn: one objective per PR.
- Do not leave placeholder text in final changes or PR descriptions.
- Fill PR sections with concrete, reviewable details:
  - Description of changes
  - GitHub issues resolved by this PR
  - Quality Assurance
- For package/versioned release changes, include a `.changeset/` entry.
- Run relevant checks before requesting review and include outputs in the PR body.
